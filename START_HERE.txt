🚀 HOW TO GET STARTED
════════════════════════════════════════════════════════════════════════════

STEP 1: Install Dependencies
  $ pip install -r requirements.txt

STEP 2: Initialize Database (First Time Only)
  $ python setup.py
  → Choose MySQL or PostgreSQL
  → Confirms data files are present

STEP 3: Run the Framework
  $ python orchestrator.py
  → Opens interactive menu
  → Select pipeline (1-5)
  → Monitor execution
  → View reports

STEP 4: Test (Optional)
  $ python test_parser.py
  $ python tests/parser_smoke_test.py


📋 MENU OPTIONS IN ORCHESTRATOR.PY
════════════════════════════════════════════════════════════════════════════

💾 DATABASE SCHEMA
════════════════════════════════════════════════════════════════════════════

execution_metadata table:
  - Tracks all pipeline executions
  - Stores: run_id, pipeline_name, total_records, execution_time_ms, etc.

daily_traffic_summary table:
  - Query 1 results
  - Columns: log_date, status_code, request_count, total_bytes

top_resources table:
  - Query 2 results
  - Columns: resource_path, request_count, total_bytes, distinct_host_count

hourly_error_analysis table:
  - Query 3 results
  - Columns: log_date, log_hour, error_request_count, total_request_count, 
             error_rate, distinct_error_hosts


📊 BATCH PROCESSING
════════════════════════════════════════════════════════════════════════════

• Configurable batch sizes: 1000, 5000, 10000 records
• Sequential batch IDs starting from 1
• Malformed records tracked separately
• Average batch size calculated automatically
• Each batch processed independently


🔄 PIPELINE EXECUTION FLOW
════════════════════════════════════════════════════════════════════════════

1. User selects pipeline in orchestrator
2. Framework reads configuration (batch size, database, data file)
3. Parser opens NASA log file
4. Records read in batches
5. Each batch aggregated using QueryProcessor
6. Results stored in database
7. Execution metadata recorded
8. Report generated on completion
